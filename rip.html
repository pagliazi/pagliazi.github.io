动态路由学习记录--RIP

By Zayl on Oct 20, 2018

路由信息协议 RIP (Routing Information Protocal)
距离向量(Distance-Vector)型路由协议

默认管理距离120，处于UDP上层，所接收路由信息都封装在UDP数据报中，使用UDP 520端口发送接受路由消息，先更新本地路由表，再通知其他路由器

  1. 使用Hop count作为路径选择度量值（metric代价）
  2. 最大跳数15

    3. 超过15则认为这条路由失效

  3. 周期性的广播或多点传送整个路由表

    4. 默认30s更新

拓扑变化

拓扑变化时，RIP处理过程

  1. R1拓扑发生变化
  2. R1更新自己的路由表
  3. R1发送更新后的路由表给相邻路由器R2
  4. R2更新自己的路由表
  5. R2发送更新后的路由表给相邻路由器R3

RIP定时器

Sending updates every 30 seconds, next due in 28 seconds
Invalid after 180 seconds, hold down 180, flushed after 240
Outgoing update filter list for all interfaces is not set
Incoming update filter list for all interfaces is not set
  • 路由更新（Update）定时器： 设置定期路由更新的时间间隔
  • 路由失效（Invalid）定时器： 路由器认定一个路由无效前的等待时间
  • 抑制（Hold down）定时器： 设置路由信息被抑制的时间
  • 路由刷新（Flushed）定时器： 将路由设为无效，并从路由表中删除的时间间隔

RIP配置

如图配置完接口后输入：

R1(config)#router rip
R1(config-router)#network 1.0.0.0

配值RIP，启动路由选择进程

network命令允许路由选择进程决定哪些接口参与发送接收路由选择更新信息，能在路由器上所有位于特定网络内的接口上启动路由选择协议
